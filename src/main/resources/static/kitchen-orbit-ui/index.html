Kitchen Orbit / Cook RAG Frontend

把菜谱问答、探索检索和下一阶段 GraphRAG 入口先落成一个前端壳层。

这一版先实现两条核心链路：自然语言问答交互页、食谱检索浏览页。页面默认支持本地 mock， 后续可直接改为对接 FastAPI 接口。

Static UI Ready

建议下一步把 `/api/chat` 和 `/api/search` 接进来，再补用户画像、图谱探索和图片上传入口。

01 / Ask

问答交互页

打开页面

用于承载“今天吃什么”“这道菜怎么做”“有什么清淡晚餐推荐”这类自然语言问答。界面保留会话历史、推荐问题、回答状态和引用卡位。

A1

会话历史

保留上下文、支持快捷追问与重新提问。

A2

推荐提问

为用户提供起始问题和预设场景入口。

A3

接口预留

支持本地 mock 和后续 `/api/chat` 切换。

02 / Search

食谱检索页

打开页面

用于分类浏览、关键词检索、时长和难度过滤，后续可作为混合检索、图谱探索、多模态检索的统一入口。

浏览模式 分类 + 筛选 + 搜索
后续扩展 图谱关系推荐 / 图片搜菜
接口规划 `/api/search` + `/api/recommend`

Tech Stack

前端技术

  • HTML5 页面结构
  • JavaScript 负责状态与 mock 接口
  • Tailwind CSS + 自定义 CSS 变量构建视觉层

Routing

页面结构

  • `index.html` 作为入口与导航页
  • `qa.html` 承载问答交互
  • `search.html` 承载食谱检索

API Ready

对接策略

  • 默认使用内置示例数据保证页面可演示
  • 提供 `window.KITCHEN_ORBIT_CONFIG` 作为接口配置入口
  • 后续接入 FastAPI 时无需重写页面结构
